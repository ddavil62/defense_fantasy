# 🏰 판타지 타워 디펜스 — Game Design Document v0.1

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 판타지 타워 디펜스 (Tower Defense) |
| 테마 | 마법, 엘프, 드래곤이 등장하는 판타지 세계관 |
| 진행 방식 | 무한 웨이브 (Endless Wave) — 얼마나 오래 버티느냐가 목표 |
| 플랫폼 | HTML5/JS 개발 → Capacitor로 iOS/Android 모바일 패키징 |
| 맵 구성 | 고정 맵 1개 (단일 맵에 집중하여 퀄리티 확보) |
| 영웅 시스템 | 없음 — 타워 배치와 전략으로만 승부 |

---

## 2. 핵심 게임플레이

### 2-1. 기본 흐름

1. 게임 시작 시 초기 Gold를 지급받아 타워를 배치한다.
2. 웨이브가 시작되면 적이 정해진 경로를 따라 이동하며 기지를 향해 전진한다.
3. 적이 기지에 도달하면 기지 HP가 감소한다.
4. 기지 HP가 0이 되면 게임 오버. 도달한 라운드 수를 기록한다.
5. 세션 종료 후 도달 라운드에 비례해 Diamond를 획득, 영구 업그레이드에 사용한다.

### 2-2. 난이도 곡선

점진적 강화 + 10라운드마다 미니보스 허들의 조합으로 설계한다.

| 구간 | 난이도 패턴 | 특이사항 |
|------|------------|---------|
| R1 ~ R9 | 완만한 상승 | 튜토리얼 구간, 기본 적 위주 |
| R10 | 미니 보스 등장 | 구간 허들 — 갑작스러운 난이도 스파이크 |
| R11 ~ R19 | 다시 완만한 상승 | 새로운 적 유형 추가 도입 |
| R20 | 미니 보스 등장 | R10보다 강한 보스 |
| R10n (반복) | 미니 보스 반복 | 10라운드마다 허들, 보스 강도는 계속 증가 |
| R100+ | 극한 난이도 | 복합 적 조합, 대량 스폰 |

## 3. 자원 시스템

### 3-1. Gold (인게임 화폐)

타워 설치, 업그레이드, 판매에 사용하는 실시간 자원.

| 획득 방법 | 상세 |
|----------|------|
| 적 처치 | 적 종류별 Gold 차등 지급 (일반 1~10G, 보스 50~200G) |
| 웨이브 클리어 보너스 | 웨이브 종료 시 잔여 기지 HP 비례 보너스 지급 |
| 타워 판매 | 설치 비용의 60% 환급 |

### 3-2. Diamond (메타 화폐)

세션 종료(게임 오버) 시 도달 라운드 수에 비례하여 획득. 앱을 껐다 켜도 유지되는 영구 자원.

| 사용처 | 설명 |
|--------|------|
| 영구 스탯 업그레이드 | 전체 타워 공격력/사거리 강화, 초기 Gold 증가 등 |
| 타워 해금 | 기본 제공되지 않는 고급 타워를 Diamond로 영구 해금 |
| 기지 HP 강화 | 기지 최대 HP 영구 증가 |
| Diamond 효율 업 | 라운드당 Diamond 획득량 증가 |

## 4. 타워 시스템

### 4-1. 기본 타워 10종

| 타워명 | 공격 타입 | 특징 |
|--------|----------|------|
| 궁수 타워 | 단일 원거리 | 빠른 공격 속도, 기본 딜러 |
| 마법사 타워 | 광역 폭발 | 느리지만 범위 피해, 중반 핵심 |
| 얼음 타워 | 단일 + 슬로우 | 적 이동속도 감소, 딜 타워와 시너지 |
| 번개 타워 | 연쇄 공격 | 최대 4~5 적에게 번개 연쇄 |
| 불꽃 타워 | 도트 대미지 | 화상 효과, 시간당 지속 피해 |
| 바위 타워 | 단일 강타 | 느리지만 높은 HP 적에게 특효 |
| 독 안개 타워 | 광역 디버프 | 범위 내 방어력 감소 + 독 피해 |
| 바람 타워 | 밀치기 | 적을 경로 뒤로 밀어내 시간 벌기 |
| 빛 타워 | 관통 | 일직선으로 다수 적 관통 |
| 드래곤 타워 | 광역 브레스 | 최고 성능, Diamond로 해금 필요 |

### 4-2. 업그레이드 트리 구조

모든 타워는 3단계 분기 업그레이드를 지원한다. 매 단계에서 2가지 방향 중 하나를 선택하며, 선택에 따라 타워의 성격이 완전히 달라진다. 목표 총 타워 수는 **30종**.

| 단계 | 구조 | 설명 |
|------|------|------|
| Lv.1 | 기본 타워 10종 | Gold로 설치, 기본 성능 |
| Lv.2 | A 또는 B 선택 (20종) | 공격 특화 vs 유틸 특화 방향 분기 |
| Lv.3 | 일부 추가 분기 (총 30종) | 전문화 완성 — 타워 정체성 확정 |

**예시 — 궁수 타워 분기:**
```
궁수 타워 (Lv.1)
├── Lv.2-A: 폭발 화살 (소형 광역 추가)
│   ├── Lv.3-A: 연속 폭발
│   └── Lv.3-B: 화염 화살
└── Lv.2-B: 저격 (사거리+, 단일 치명타)
    └── Lv.3: 저격 특화 (치명타율 대폭 증가)
```

## 5. 적(Enemy) 시스템

### 5-1. 적 유형 분류

| 유형 | 특성 | 대응 전략 |
|------|------|----------|
| 일반형 | 표준 속도·HP | 기본 타워 배치로 처리 |
| 고속형 | 이동속도 매우 빠름, 낮은 HP | 얼음·바람 타워로 속도 제어 |
| 탱커형 | 매우 높은 HP, 느린 속도 | 바위·불꽃(도트) 집중 |
| 소형 군집형 | 크기 작고 수가 많음 | 광역 타워(마법사·독) 필수 |
| 대형 보스형 | 10라운드마다 등장, 고HP + 특수 패턴 | 전체 타워 집중 + 자원 비축 |
| 분열형 (어머니) | 처치 시 소형 적 2~4마리 소환 | 광역 타워로 연쇄 처리 |
| 방어형 | 특정 속성 저항 보유 | 저항 없는 타워 타입으로 대응 |

### 5-2. 분열형 (어머니) 적 상세

처치 시 자신의 몸에서 소형 적을 소환하는 특수 유형. 단일 타깃 타워만으론 처리가 어려워 광역 타워와의 조합이 필수적이다.

- 소형 적의 수: 2~4마리 (라운드가 높을수록 증가)
- 소형 적은 어머니보다 빠르지만 HP가 낮음
- 미니 보스 중 일부를 분열형으로 설계하여 극적인 장면 연출

## 6. 개발 로드맵

| Phase | 목표 | 주요 작업 |
|-------|------|----------|
| Phase 1 — 프로토타입 | 핵심 루프 구현 | 맵 렌더링, 적 경로 이동, 기본 타워 3종, 웨이브 시스템 |
| Phase 2 — 콘텐츠 1차 | 기본 타워 10종 완성 | Gold 시스템, 업그레이드 Lv.2 분기, 적 유형 5종 |
| Phase 3 — 메타 시스템 | Diamond 루프 완성 | 세션 종료 보상, 영구 업그레이드 UI, 타워 해금 시스템 |
| Phase 4 — 게임플레이 완성 | Lv.3 분기 + UX | Lv.3 업그레이드 분기 완성(30종), 게임 내 일시정지/메뉴 UI |
| Phase 5 — 보스 & 사운드 | 연출 강화 | 보스 패턴(특수 행동), 효과음/BGM, 전투 연출 폴리싱 |
| Phase 6 — 패키징 & 최종 | 출시 준비 | Capacitor 모바일 패키징, 밸런스 패치, 최종 폴리싱 |

## 7. 기술 스택

| 분야 | 기술 |
|------|------|
| 게임 엔진 | Phaser.js 3 (HTML5 Canvas 기반 2D 게임 프레임워크) |
| 언어 | JavaScript (ES6+) 또는 TypeScript |
| 모바일 패키징 | Capacitor (iOS / Android 동시 대응) |
| 데이터 저장 | LocalStorage — 라운드 기록, Diamond, 영구 업그레이드 |
| 에셋 | 오픈소스 스프라이트 활용 (itch.io, OpenGameArt.org) |
