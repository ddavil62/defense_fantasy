# 경제 시스템

Gold (인게임 화폐) + Diamond (메타 화폐) + 메타 업그레이드 + 컬렉션 모드 + 골드 싱크.

## Gold 시스템

- 초기: 160G (글로벌 시작 골드 업그레이드 적용 시 최대 360G = 160 + 20*10)
- 적 처치 시 타입별 보상 (Normal 5G, Fast 3G, Tank 15G, Boss 100G, Swarm 2G, Splitter 12G, Armored 20G)
  - 배율 적용: `enemy.gold * killGoldMult * goldRainMultiplier * adGoldMultiplier` (글로벌 처치보상 ×1.1^n, Gold Rain 소모품 2x, 광고 골드 부스트 2x)
- 웨이브 클리어 보너스: `round * 3 + floor(currentHP / maxHP * 8)`
  - 배율 적용: `bonusGold * globalWaveBonusMult * adGoldMultiplier` (글로벌 웨이브 보너스 ×1.1^n, 광고 골드 부스트 2x)
- 소비: 타워 뽑기 배치, 업그레이드 (Lv.2, Lv.3), 타워 강화, HP 회복, 소모품 능력, 합성 비용
- **타워 뽑기 비용**: `floor((80 + 20 * drawCount) * drawDiscountMult)` (글로벌 뽑기 할인 ×0.95^n 적용, 배치 확정 시에만 차감)
  - 0회차: 80G, 1회차: 100G, 5회차: 180G, 10회차: 280G, 20회차: 480G
  - 개별 타워 가격(`TOWER_STATS[type].levels[1].cost`)은 배치 시 사용되지 않음 (판매 환급 계산에는 여전히 사용)
- **광고 보상 타워**: 광고 시청 후 T1(90%)/T2(10%) 타워 3개 중 1개를 무료 배치. 골드 소모 없음, drawCount 미증가, 횟수 제한 없음
  - T1 타워: cost=0, `totalInvested=0`
  - T2 타워: cost=0, `totalInvested=0`, base 타워 생성 후 `applyMergeResult()` 적용
  - 판매 시 환급: 0G (`totalInvested * SELL_RATIO = 0`)
- 판매 환급: 총 투자 비용의 50% (`SELL_RATIO = 0.5`)
- 타워 배치 한도: 최대 30기 동시 배치 (`MAX_TOWER_COUNT = 30`)
- 합성 골드 비용: T2=30G, T3=80G, T4=150G, T5=250G (`MERGE_COST`, 글로벌 합성 할인 ×0.95^n 적용)

## 기지 HP 시스템

- 초기 HP: 20 (글로벌 기지 HP 업그레이드 적용 시 최대 50 = 20 + 3*10)
- HP 회복: 골드로 5 HP 회복 가능 (골드 싱크 시스템 참조)
- HP <= 5 시 빨간 깜빡임
- 적 기지 통과 시 빨간 플래시 이펙트
- HP 0 → 게임 오버

## Diamond 메타 화폐 (Phase 3)

### 획득 공식

```
floor(round / 5) * 2 + floor(round / 10) * 3
```

| 도달 라운드 | Diamond 획득 |
|---|---|
| R5 | 2D |
| R10 | 7D |
| R15 | 6D |
| R20 | 11D |
| R30 | 18D |

- 게임 오버 시 획득, 영구 축적
- 게임 오버 화면에 획득량/총량 표시
- 보상형 광고 시청: MenuScene "Diamond 받기" 버튼으로 1회당 3개 지급 (일일 5회 제한, `AD_REWARD_DIAMOND`/`AD_LIMIT_DIAMOND`)
- 캠페인 클리어 보상 2배: LevelSelectScene/MapClearScene "보상 2배" 광고 시청으로 다이아몬드 차액 보상 2배 (일일 3회 제한, `AD_CLEAR_BOOST_MULTIPLIER`/`AD_LIMIT_CLEAR_BOOST`)

### 소비처

- 글로벌 메타 업그레이드 (컬렉션 모드)

## 컬렉션 모드

메뉴 화면의 COLLECTION 버튼으로 진입.

### 글로벌 메타 업그레이드 (10종)

카테고리별로 분류된 영구 업그레이드. Diamond를 소모하여 레벨을 올린다. 모든 타워/게임 전반에 적용.

- **비용 공식**: `5 + currentLevel * 3` Diamond (Lv0→1: 5D, Lv1→2: 8D, Lv9→10: 32D)
- **풀강화 누적 비용**: 10레벨 기준 185D, 5레벨(사거리) 기준 55D
- **전체 풀강화 총 비용**: 185D × 9 + 55D = 1,720D

| 공격력 | ⚔ | ×1.1^n | 10 | ×2.594 |
| 공격속도 | ⚡ | ×0.9^n (간격 감소) | 10 | ×0.349 |

### 적용 위치

- **공격력/공격속도/사거리**: `GameScene._applyGlobalUpgradesToTower()` — 타워 배치/업그레이드 시
- **시작 골드**: `GameScene.create()` — 게임 시작 시
- **처치 보상**: `GameScene` — 적 처치 시 골드 계산
- **뽑기 할인**: `TowerPanel` — 뽑기 비용 계산
- **합성 할인**: `TowerPanel`, `GameScene` — 합성 비용 계산
- **기지 HP**: `GameScene.create()` — 게임 시작 시
- **웨이브 보너스**: `GameScene` — 웨이브 클리어 보너스 계산
- **다이아 보너스**: `GameOverScene`, `MapClearScene` — Diamond 획득 시

- `migrateSaveData()` 함수로 순차 마이그레이션 (v1 → v2 → ... → v9)

## 광고제거 인앱 구매 (IAP)

`@capgo/native-purchases@^7.16.2` 플러그인을 통해 Google Play Billing에 직접 연동한다.

### 상품 정보

| 항목 | 값 |
|------|-----|
| 상품 ID | `remove_ads` (config.js `IAP_PRODUCT_REMOVE_ADS`) |
| 유형 | 비소모품 (일회성) |
| 기본 가격 | $1.99 USD |
| 효과 | 모든 광고 제거 + 무료뽑기 무제한 + Diamond 받기 무제한 |

### 구현 구조

- **IAPManager 클래스** (`js/managers/IAPManager.js`): Mock/Native 이중 구현
  - 네이티브: 동적 import로 `@capgo/native-purchases` 로드, Google Play Billing API 호출
  - 웹: Mock 모드로 즉시 성공 처리
- **초기화**: `isBillingSupported()` + `getProducts()` (상품 정보 캐싱)
- **구매**: `purchaseProduct()` (성공 시 `saveData.adFree = true`, AdManager 동기화)
- **복원**: `getPurchases()` (구매 이력에서 `remove_ads` 존재 확인)
- **가격 조회**: `getLocalizedPrice()` (캐싱된 상품 정보에서 현지화 가격, 폴백 `$1.99`)
- **결제 지원 확인**: `isBillingSupportedSync()` (Mock에서 항상 true)

### MenuScene 연동

- 광고제거 버튼에 `getLocalizedPrice()`로 동적 가격 표시
- `isBillingSupportedSync()` false 시 버튼 비활성화 + "결제를 지원하지 않는 기기입니다" 토스트
- 구매 완료 시 버튼 비활성화 ("구매완료" 표시)

### 에러 처리

| 상황 | 처리 |
|------|------|
| 결제 지원 안됨 | 구매 버튼 비활성화, 토스트 표시 |
| 사용자 취소 | 조용히 무시 (cancel/user 키워드 감지) |
| 네트워크 오류 | "구매 실패" 메시지 표시 |
| 이미 구매됨 | adFree 자동 적용 (already/owned 키워드 감지) |
| 상품 로드 실패 | 폴백 가격($1.99) 표시, 구매 시도 허용 |
| 초기화 실패 | Mock 모드로 자동 폴백 |

### 알려진 제약

- Google Play Console에 `remove_ads` 인앱 상품 등록이 선행되어야 네이티브 가격 표시 및 실결제 가능
- 취소 판별 시 "user" 키워드 false positive 가능성 (LOW 심각도, 운영 모니터링 후 개선 판단)

## 골드 싱크 시스템

후반 라운드(R30+)에서 축적되는 골드 인플레이션을 해소하기 위한 3가지 골드 소비처.

### 1. 타워 강화

Lv.3 타워에 추가 골드를 투자하여 스탯을 강화한다. 최대 10단계.

- **대상**: Lv.3 이상 타워만 강화 가능
- **최대 강화 레벨**: 10
- **비용 공식**: `200 + 100 * currentEnhanceLevel` (에스컬레이팅)
- **강화당 스탯 보너스**:
  - 공격력: x1.05 (+5%)
  - 사거리: x1.025 (+2.5%)
  - 공속: x0.975 (-2.5%, 최소 0.3초)
- **시각 효과**: 강화된 타워에 금색 글로우 링 표시
- **UI**: 타워 이름에 `+N` 표시 (예: "궁수 Lv.3AA +5")

| 강화 레벨 | 비용 | 누적 투자 |
|---|---|---|
| +1 | 200G | 200G |
| +2 | 300G | 500G |
| +3 | 400G | 900G |
| +4 | 500G | 1,400G |
| +5 | 600G | 2,000G |
| +6 | 700G | 2,700G |
| +7 | 800G | 3,500G |
| +8 | 900G | 4,400G |
| +9 | 1,000G | 5,400G |
| +10 | 1,100G | 6,500G |

타워 1기 풀강화에 6,500G 소요. R53 기준 45,000G에서 타워 6~8기 풀강화(39,000~52,000G)로 대부분 소진 가능.

### 2. 기지 HP 회복

골드를 소비하여 기지 HP를 회복한다.

- **회복량**: 5 HP/회
- **비용 공식**: `100 + 50 * useCount` (에스컬레이팅)
- **HP 최대일 때**: 사용 불가 ("HP 최대" 표시)
- **연출**: 초록 플래시 이펙트

| 사용 횟수 | 비용 | 누적 비용 |
|---|---|---|
| 1회차 | 100G | 100G |
| 2회차 | 150G | 250G |
| 3회차 | 200G | 450G |
| 5회차 | 300G | 1,000G |
| 10회차 | 550G | 3,250G |

### 3. 소모품 능력

3종의 액티브 능력을 골드로 사용한다. 각 능력은 쿨다운과 에스컬레이팅 비용을 가진다.

| 능력 | 버튼 | 효과 | 쿨다운 | 초기 비용 | 비용 증가 |
|---|---|---|---|---|---|
| 슬로우 | S | 전체 적에게 `applySlow()` 적용 (5초, 50%) | 30초 | 300G | +50G/회 |
| 황금비 | $ | 10초간 적 처치 골드 2배 | 45초 | 400G | +75G/회 |
| 번개 | Z | 전체 적에게 100 피해 + 카메라 흔들림 (즉시) | 60초 | 500G | +100G/회 |

#### 소모품 세부 규칙

- **쿨다운**: 사용 후 지정 시간 동안 재사용 불가. 쿨다운 중 버튼에 잔여 시간 표시
- **적 없을 때 사용 제한**: 즉시형 능력(슬로우, 번개)은 살아있는 적(`alive && !reachedBase`)이 없으면 사용 불가. 지속형 능력(황금비)은 사전 활성화 허용
- **비용 에스컬레이팅**: 사용할 때마다 비용이 증가하여 무한 사용 방지
- **연출**: 능력 사용 시 화면 플래시 연출 (`_playAbilityFlash`)

#### 소모품 최대 투자량 (10회 기준)

| 능력 | 10회 누적 비용 |
|---|---|
| 슬로우 | 5,250G |
| 황금비 | 7,375G |
| 번개 | 9,500G |
